# TreeK Pro Features

## Tree Options And Settings In TreeK Pro

### Tree Creation

- Display the post tree: each post is represented by a row in the tree.
- A post created with the Reply button is treated as a child post; the post being replied to is treated as its parent.
- The relationship between a child row and its parent row is shown visually by shifting the child row to the right.

### Interaction Between Kunena And TreeK

- Open the topic tree from the topics list, for example from Kunena's Recent Topics tab, by clicking the Replies number.
- Open the topic tree from any post in a topic by clicking the tree button at the end of the post header. The tree opens on the corresponding post row.
- Return from the tree to the Kunena topic by closing the tree window.
- Jump directly from a tree row to the corresponding post in the Kunena topic.

### Tree Window Settings And Options

- Use the tree settings window.
- Move the settings window by dragging it.
- Use different grid styles to emphasize the tree structure.
- Use vertical and horizontal scrolling for large trees.
- Keep the topic title visible in the sticky header of the tree window.
- Show tooltips for TreeK buttons and symbols.
- Use the flat timeline list view, which follows the Kunena post order.
- Save and restore settings.
- Export the tree as BBCode, HTML, or text.
- Apply tree transformations immediately when a setting is changed.
- Show a topic monitoring notice when a new post is added. This is a unique option that is absent from existing forum programs.

### Tree Row Display Settings

- Start a row with Subject + Author, Author + Subject, or Subject.
- Show date and time.
- Show post index.

### Showing Post Content In Tree Rows

- Show post text as a tooltip.
- Show the beginnings of all post texts under the tree rows with Text teaser.
- Show the real visual appearance of the top parts of all posts under the tree rows with Screenshot teaser.

Screenshot teaser makes it possible to:

- see images and other non-text elements at the beginning of posts;
- organize a full forum with indented post structure.

### Highlighting Important Rows With Comfort Buttons

- Highlight the row of the latest post created in the topic, even if it is located in the middle of the tree.
- Highlight active rows, meaning rows currently involved in the user's work.
- Highlight all rows/posts by a specific author in the tree.
- Highlight all child rows of a specific post, if they exist.
- Highlight the parent row of a specific post.

### Tree Navigation With Comfort Buttons

- Jump from a post row to any of its child rows.
- Jump from a post row to its parent row.

### Additional Kunena User Interface Comfort Features

These options make the tree more expressive and can slightly speed up forum work.

In post creation and editing forms:

- The post subject can be written freely so that it reflects the meaning of the post.
- The topic title does not change because of freely chosen post subjects.

In topic posts:

- A parent navigation button is added at the beginning of a post header.
- Comfort form header clears the subject field so that the user can enter a meaningful custom post subject.
- Subject suffixes provide an advanced system for quickly forming post subjects.
- Collapsed attachments hides the attachments area when it is not used.
- 3 action buttons restores the traditional Reply, Quote, and Edit buttons instead of the Action dropdown menu.
